## Git 運用ルール

### 基本方針

- コードを変更するたびに、変更内容をコミットしてGitHubにプッシュすること
- コミットは作業の区切りごとに行い、意味のある単位でまとめること

### コミット手順

1. 変更内容を確認: `git status` / `git diff`
2. 変更ファイルをステージング: `git add <ファイル名>`（`git add .` は使わない）
3. コミット作成（日本語メッセージ可）
4. GitHubへプッシュ: `git push origin <ブランチ名>`

### コミットメッセージ

- 変更の「なぜ」を簡潔に記述する
- 1〜2文程度に収める
- Co-Authored-By フッターを付ける:
  ```
  Co-Authored-By: Claude Sonnet 4.6 <noreply@anthropic.com>
  ```

### 禁止事項

- `--no-verify` でフックをスキップしない
- `git push --force` を main/master ブランチに対して使わない
- `.env` や認証情報を含むファイルをコミットしない
- `git add -A` や `git add .` で不要なファイルを一括ステージングしない

### ブランチ戦略

- 通常作業は `main` ブランチで行う
- 大きな変更や実験的な作業は専用ブランチを切る

## 作業スタイル

- コメントは「なぜ」が非自明な場合のみ記述する
- 不要な抽象化・機能追加はしない
- セキュリティに関わるコード（入力検証、認証等）は特に慎重に実装する
